# Итерации NER-экстрактора

Логика разработки: `app/nlp/extractor.py` собирает теги через SpaCy NER +
KeyBERT, потом фильтрует. Качество глазами проверяем на фиксированной
выборке из 5 персон (см. `scripts/test_extractor.py` → `SAMPLE_PERSON_IDS`).

Полные вывода — рядом: `extractor_v1_sample.txt`, `extractor_v2_sample.txt`.

## v1 — baseline (без post-processing)

Вход: текст из `build_person_context` (имя + интересы + bio + work_experience +
30 последних публикаций с абстрактами).

Логика: SpaCy `ru_core_news_lg` / `en_core_web_sm` для NER (типы
`ORG, PRODUCT, WORK_OF_ART, LAW, EVENT, NORP`) + `noun_chunks` (en) +
KeyBERT `paraphrase-multilingual-MiniLM-L12-v2` (n-gram 1–3, MMR, diversity=0.5,
top_n=30). Объединение, нормализация, удаление стоп-слов, дедуп по подстроке.

### Проблемы baseline

1. **Названия организаций как теги** — `минобрнауки`, `росстата`,
   `банка россии`, `правительстве рф`, `национального исследовательского
   университета`, `центра прикладных экономических исследований и разработок`.
   SpaCy ORG отбирает их как кандидатов, KeyBERT тоже.
2. **Имя/фамилия персоны попадает в теги** — `сфере образования абанкина`.
3. **Кафедры / факультеты / институты** — `кафедры региональная`,
   `высшая школа урбанистики им. а.а. высоковского`,
   `институт статистических исследований и экономики знаний`.
4. **Академический boilerplate** — `научно педагогический стаж`,
   `опыт работы`.
5. **Хвосты с датами** — `вшэ 2021 2023`, `гг ходе специализированных`.
6. **Сокращения / служебные** — `сопс`, `вавт минэкономразвития`,
   `др сборнике`.

## v2 — post-processing apply_filters

KeyBERT / SpaCy не трогали. Добавили 9 правил в `apply_filters(tags,
person_name, ner_rejects)` (порядок ≈ от дешёвых отсевов к дорогим):

1. Нормализация (lowercase, strip пунктуации) — уже было в `_normalize`.
2. **Имя персоны**: токены `full_name` длиннее 3 симв; тег, содержащий
   любой как подстроку — отсев.
3. **ORG_INDICATORS** (`университ`, `институт`, `школ`, `центр`, `комитет`,
   `министерств`, `росстат`, `минобрнауки`, `банк рос`, `ниу`, `рф`, и т.д.
   ~30 корней + сокращений; для токенов <4 симв — word-boundary, чтобы
   не было false positive типа «рана» содержит «ран»). **Высший приоритет.**
4. **SpaCy reject-типы**: ORG исключён из извлечения; ORG/GPE/LOC/DATE
   собираются в `ner_rejects`, тег, содержащий любой как подстроку — отсев.
5. **Служебные префиксы**: первое слово короче 3 симв ИЛИ ∈ `{в, с, на, по,
   при, для, гг, г, ходе, ...}` — отсев целиком.
6. **Числовой коэффициент**: доля цифр + пробелов + знаков `.,–-/` + `гг/г.`
   ≥ 40% — отсев.
7. **JUNK_PHRASES**: подстрочное совпадение с шаблоном (`научно
   педагогический стаж`, `опыт работы`, `публикуется в`, ...) — отсев.
8. Существующая дедупликация по подстроке (длинный поглощает короткий).
9. Минимальная длина 4 симв + хотя бы одно слово ≥ 4 симв.

### Результат v2

- Все названия орг и кафедр исчезли (см. дифф v1 → v2 в README сессии).
- Имя персоны исчезло.
- Boilerplate исчез.
- Абашкин теперь имеет `кластерная политика региональная`,
  `инновационная политика кластерная` — реальные интересы.
- Абдуллаев — `пространственное развитие городов`, `социально экономическая
  география`.

### Что осталось (вне спеки v2)

- Теги, начинающиеся с 4-значного года (`2001 году научно`, `2020 лаборант
  исследователь`) — numeric ratio < 40%, проходят. Можно добавить отдельный
  фильтр «первый токен — год».
- `стаж 16 лет`, `стаж 20 лет` — JUNK_PHRASES не содержит «стаж» как
  подстроку. Можно расширить.
- Шумные фразы типа `результаты не`, `data book` — низкосодержательные,
  но не явный мусор.
