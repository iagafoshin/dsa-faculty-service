## DOM variant playbook (what the refactored parser must target)

Per-section structural notes gathered by manually inspecting representative
fixtures. When a section has multiple variants, the parser should try them in
order and fall back to the next on empty result.

### Positions (`ul.employment-add li`)

Each `<li>` has a `<span class="person-appointment-title">` with a single
string that may join multiple role titles with commas and end with ":":

```
<span class="person-appointment-title">руководитель департамента, профессор: </span>
```

Units (department links) follow as `<a class="link">` elements, separated by
`/`. Some `<li>` have no appointment title (e.g. `class="i"` items like
"Начала работать в НИУ ВШЭ в 2001 году.") — those are bio notes, not
positions, and should be excluded.

**Normalizer**: split the title on `,`, strip trailing `:`, produce one
Position per title sharing the same `units[]`.

### Work experience (`div[@tab-node="experience"]`)

One or more `.with-indent` blocks. Inside: either

- Variant A — one `<p class="text">` per entry:
  `<p>1997 - специалист, ПГАСА(ПГУАС), факультет менеджмента, 1997 г.,...</p>`
- Variant B — a single `.with-indent` whose `text_content()` concatenates
  all entries (current parser sees this). Entries inside this blob are
  identified by year patterns like `YYYY г.` or `YYYY – YYYY`.

**Normalizer**: regex-split on year patterns
(`\d{4}\s*(?:г\.|–|-|\s+по\s+н\.?в\.?)`) and pair each year range with the
following text chunk.

### Conferences (`div[@tab-node="conferences"]//li[contains(@class,"li2")]`)

Each `<li class="li2">` has an optional `<div class="person-list-hangover">`
with the year and a `<p>` with body text. Body format:

```
Title of conference («Quoted»?) (Location). Доклад: Talk title
```

Missing year → inherits `last_year` (year-grouped list). Location and talk
are both optional.

**Normalizer**: primary regex
`^(\d{4}):\s*(.+?)\s*\((.+?)\)\.\s*Доклад:\s*(.+)$` with fallbacks for
missing location or missing talk.

### Patents (`div[@tab-node="patents"]`, 6/22)

Rendered as `<table class="patent_table">` with rows having `<td>` cells
bearing `data-title` attributes. Column order observed:

| data-title | meaning |
|---|---|
| `Номер РИД` | patent number (often a link to the patent PDF) |
| `Вид РИД` | kind (Изобретение, Полезная модель, Программа для ЭВМ, …) |
| `Наименование РИД` | title |
| `Сведения о регистрации` | registration info (e.g. `US10275418B2`) |
| `Авторы` | comma-separated author short names |

**Parser**: per row, collect one dict per `td[data-title]` keyed by a
normalized English name (`number`, `kind`, `title`, `registration`,
`authors`). Year is not directly present and is optional.

### Grants — two DOM variants

- **Variant A** — tab-node present (3/22): `div[@tab-node="grants"]` with a
  `<ul>/<ol>` of `<li>` entries. Each `<li>` is free-form Russian text
  containing project title, role, grant number and a year range like
  `(2023–2026 гг.)`.
- **Variant B** — heading only (4/22): `<h2>Гранты</h2>` followed by
  `div.with-indent` with `<p class="text">` entries (e.g.
  `education_papushina`). Also seen as `<h2>Исследовательские проекты и
  гранты</h2>` (`math_lepskiy`).

**Parser**: try tab-node first, then fall back to heading-based XPath
(`//h2[contains(text(),"ранты")]/following-sibling::div[1]//p[@class="text"]`).

### Awards (`div[@tab-node="awards"]`, 21/22)

`ul.g-list li` with free-form text. Optional trailing year range in
parentheses: `Медаль "За вклад..." НИУ ВШЭ (февраль 2025)`, or ranges
like `(2025–2026)`.

**Normalizer**: parse optional year range with
`\((?:[^\d()]*?)(\d{4})(?:\s*[–\-]\s*(\d{4}))?\s*(?:г\.|гг\.)?\)` from the
end of the string.

### Interests (`div[@tab-node="sci-intrests"]`, 20/22)

`a.tag` anchors. Simple list of labels. `possibly_foreign` and
`staff_slug_aspirant_with_pubs` have no interests tab — return empty list.

### Editorial staff (`div[@tab-node="editorial-staff"]`, 6/22)

`.with-indent` blocks with free-form text. Year pattern `YYYY г.` and
journal name in French quotes `«...»` often present.

### Education — degrees (`div[@tab-node="sci-degrees1"]`, 22/22)

Existing parser is correct. `.with-indent` entries; year in
`div.person-list-hangover`.

### Research IDs, contacts, managers, courses, news

Current XPaths in `parser.py` match all 22 fixtures (verified via class
marker counts). Keep as-is, just encapsulate in per-section functions.

### Sections present on the page but never exercised

None of the 22 fixtures contain `tab-node=bio`, `theses`, `public_speeches`,
`d_council`, `expertise`, or `activities`. We therefore do **not** add
parsing code for them in this refactor — the JSON shape is flexible
(JSONB) and they can be added later as fixtures show up.
